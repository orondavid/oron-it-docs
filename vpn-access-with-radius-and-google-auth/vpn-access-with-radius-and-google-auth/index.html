    מדריך להתקנת VPN ב־pfSense עם חיבור ל־Active Directory ואימות MFA של Google (2FA)

    🎯 מטרת המדריך

    להגדיר חיבור VPN מאובטח ב־pfSense עם אימות כפול (2FA) באמצעות Active Directory ו־Google Authenticator, כולל הגדרות צד שרת וצד לקוח.

    דרישות מקדימות

    • pfSense מותקן ופעיל עם חיבור לאינטרנט
    • שרת Active Directory פעיל (Domain Controller)
    • שירות DNS פנימי פעיל עבור pfSense
    • טלפון נייד עם אפליקציית Google Authenticator או כל TOTP אחר
    • חבילת openvpn-client-export מותקנת ב־pfSense

    שלב 1: הגדרת חיבור ל־Active Directory (LDAP)

    1. עבור ל־System > User Manager > Authentication Servers.
    2. לחץ Add.
    3. מלא:
    4. Descriptive name: AD
    5. Type: LDAP
    6. Hostname: כתובת ה־Domain Controller (לדוגמה dc.example.local)
    7. Port: 389 או 636 (אם עם SSL)
    8. Transport: TCP - Standard או SSL
    9. Base DN: לדוגמה DC=example,DC=local
    10. Bind credentials: משתמש עם הרשאות קריאה
    11. Authentication Containers: CN=Users,DC=example,DC=local
    12. לחץ Save and Test כדי לוודא התחברות מוצלחת.

    שלב 2: התקנת FreeRADIUS והפעלת תמיכה ב־TOTP

    1. עבור ל־System > Package Manager > Available Packages.
    2. התקן את freeradius3.

    הגדרות ב־FreeRADIUS:

    1. עבור ל־Services > FreeRADIUS > Settings:
    2. סמן Enable Mobile One-Time Password Support
    3. בחר Hash: SHA256
    4. לשונית Interfaces:
    5. Listener על פורט 1812/1813
    6. לשונית Clients:
    7. הוסף את pfSense כ־Client
    8. הזן Shared Secret חזק
    9. לשונית Users:
    10. צור משתמשי בדיקה עם TOTP (לדוגמה: vpnuser Cleartext-Password := "123456", Google-Authenticator := "yes")
    11. בצד שמאל יופיע QR סרוק עם Google Authenticator

    שלב 3: הגדרת שרת OpenVPN

    1. עבור ל־VPN > OpenVPN > Wizards.
    2. צור CA חדשה ותעודת שרת.
    3. הגדר:
    4. Backend for authentication: RADIUS
    5. Tunnel Network: לדוגמה 10.8.0.0/24
    6. Local Network: הרשת הפנימית (לדוגמה 192.168.1.0/24)
    7. Redirect Gateway: לפי הצורך
    8. DNS Servers: 8.8.8.8 (או פנימי)
    9. Compression: off
    10. Port: 1194 UDP (ברירת מחדל)
    11. סיים את ההגדרה ושמור.

    שלב 4: הגדרת pfSense כ־Client ב־FreeRADIUS

    1. עבור ל־System > User Manager > Authentication Servers.
    2. הוסף שרת חדש:
    3. Type: RADIUS
    4. Hostname: 127.0.0.1
    5. Shared secret: אותו Secret שהוזן ב־FreeRADIUS
    6. Authentication port: 1812
    7. Protocol: PAP (הכרחי ל־OTP)

    שלב 5: הגדרת חומת אש (Firewall)

    1. עבור ל־Firewall > Rules > WAN.
    2. פתח פורט 1194 UDP אל כתובת ה־pfSense.

    שלב 6: ייצוא לקוח VPN והפצת קבצים למשתמשים

    1. עבור ל־VPN > OpenVPN > Client Export.
    2. הורד את קובץ ההתקנה:
    3. בחר מערכת הפעלה
    4. הורד את גרסת ה־installer או קובץ config

    📲 הפצת למשתמש הקצה:

    • יש לשלוח למשתמש את:
    • קובץ ההתקנה או הקונפיגורציה (ולוודא הצפנה בהעברה)
    • קוד QR מ־FreeRADIUS לסריקה באפליקציית Google Authenticator
    • הסבר: בעת כניסה, המשתמש יזין את הסיסמה הרגילה שלו ואחריה את קוד ה־OTP (לדוגמה: MyPassword123456)

    שלב 7: בדיקה ואימות חיבור

    1. התקן את OpenVPN Client על מחשב המשתמש.
    2. העלה את הקובץ שהורד.
    3. התחבר:
    4. שם משתמש: לפי ההגדרה
    5. סיסמה: הסיסמה + קוד OTP
    6. בדוק שה־VPN מעלה כתובת IP תקינה (לדוגמה: 10.8.0.x) ושהרשת המקומית נגישה.

    טיפים ואבטחה

    • ודא ש־NTP מסונכרן ב־pfSense וב־DC.
    • שמור סיסמאות ו־QR קודים באופן מאובטח.
    • בצע בדיקות תקופתיות למשתמשים ומחיקת ישנים.
    • תעד נסיונות כושלים (System Logs).
    • שקול הפעלת Client Specific Overrides להקצאת כתובות ייחודיות לפי משתמש.

    סיום

    מדריך זה מיישם תצורת VPN מאובטחת באמצעות pfSense ו־FreeRADIUS תוך חיבור ל־AD ואימות דו־שלבי. מומלץ לבדוק את כל השלבים מול משתמש בדיקה לפני הפצה לכלל הארגון.
