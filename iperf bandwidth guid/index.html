    📶 מדריך שימוש ב־iperf לבדיקת ביצועי רשת

    iperf היא תוכנה חינמית וקלה לשימוש לבדיקת קצב העברת נתונים (רוחב פס) בין שני מחשבים ברשת. היא מאפשרת לבדוק ביצועים בין רשתות שונות או בתוך אותה רשת מקומית (LAN).

    🔧 התקנת iperf

    לינוקס (Debian/Ubuntu):

    sudo apt update
sudo apt install iperf3

    Windows:

    1. הורד את הקובץ מ: https://iperf.fr/iperf-download.php
    2. חלץ את הקבצים.
    3. הפעל את cmd מתוך התיקיה או הגדר את הנתיב ב־PATH.

    macOS:

    brew install iperf3

    🖥️ מבנה הבדיקה

    בדיקת iperf מתבצעת בין שרת (listener) ל־לקוח (initiator).

    • מחשב אחד מריץ iperf במצב שרת
    • מחשב שני מריץ iperf במצב לקוח, ומתחבר לשרת

    📡 בדיקה בסיסית בתוך רשת מקומית

    שלב 1: הרצת iperf במחשב השרת

    iperf3 -s

    שלב 2: הרצת iperf במחשב הלקוח

    iperf3 -c 192.168.1.10

    דוגמה לפלט:

    [ ID] Interval           Transfer     Bandwidth
[  5]   0.00-10.00 sec   113 MBytes   94.5 Mbits/sec

    🛠️ בדיקות מתקדמות

    ⏱️ שינוי זמן הבדיקה (ברירת מחדל: 10 שניות)

    iperf3 -c 192.168.1.10 -t 30

    🔁 בדיקת שידור דו-כיוונית (לקוח שולח וגם מקבל בו זמנית)

    iperf3 -c 192.168.1.10 -d

    🔄 בדיקת reverse (השרת שולח ללקוח)

    iperf3 -c 192.168.1.10 -R

    ⚡ בדיקה עם מספר חיבורים במקביל (בדיקת עומסים)

    iperf3 -c 192.168.1.10 -P 5

    📁 בדיקת פורט אחר (ברירת מחדל 5201)

    שרת:

    iperf3 -s -p 5000

    לקוח:

    iperf3 -c 192.168.1.10 -p 5000

    🌐 בדיקה בין אתרים (WAN)

    תוכל לבדוק חיבור לאינטרנט באמצעות שרתי iperf ציבוריים:

    iperf3 -c iperf.he.net

    או:

    iperf3 -c bouygues.iperf.fr

    🧪 בדיקות TCP מול UDP

    בדיקת UDP (כולל אובדן חבילות ו־Jitter):

    שרת:

    iperf3 -s -u

    לקוח:

    iperf3 -c 192.168.1.10 -u -b 100M

    -b מציין את קצב השידור הרצוי (במקרה זה 100Mbps)

    📈 ניתוח תוצאות

    • Bandwidth – רוחב הפס בפועל שהושג (ב־Mbit/sec או Gbit/sec)
    • Jitter – מדד לשונות זמן בין חבילות (רלוונטי ב־UDP)
    • Lost/Total Datagrams – אחוז אובדן חבילות
    • Retransmits – חזרות שידור (רלוונטי ב־TCP)

    🧹 עצות וטיפים

    • ודא ששירותי firewall לא חוסמים את הפורטים (ברירת מחדל: 5201).
    • כדי לבדוק ביצועים אמיתיים, נסה לבדוק בכיוון הלוך וחזור (-R).
    • השתמש ב־-P עם ערך גבוה כדי לבדוק עומס רשת מרובה חיבורים.
    • תעד את תוצאות הבדיקה לקובץ:
    iperf3 -c 192.168.1.10 > result.txt

    🧭 סיכום

    פרמטר תיאור
    -s מצב שרת
    -c <IP> מצב לקוח
    -t <שניות> משך הבדיקה
    -R בדיקה בכיוון ההפוך (שרת שולח ללקוח)
    -d בדיקה דו-כיוונית
    -P <n> מספר חיבורים במקביל
    -u מצב UDP (בדיקה לא מבוססת TCP)
    -b <קצב> קצב שידור ב־UDP
    -p <פורט> שימוש בפורט מותאם אישית

    🧮 השוואת מהירויות רשת לפי סוג חיבור

    סוג חיבור / ציוד מהירות תאורטית (Mbps) מהירות בפועל (Mbps) הערות חשובות
    Ethernet 10Mbps 10 \~9 ישן מאוד, בקושי בשימוש כיום.
    Fast Ethernet (100Mbps) 100 \~94–97 מתאים רק לשימושים בסיסיים.
    Gigabit Ethernet (1Gbps) 1,000 \~940–960 הנפוץ ביותר כיום. ביצועים טובים לרוב השימושים.
    2.5G Ethernet 2,500 \~2,300–2,400 תומך בשדרוג ללא החלפת כבלים.
    5G Ethernet 5,000 \~4,600–4,800 דורש ציוד ייעודי.
    10G Ethernet 10,000 \~9,500–9,800 מתאים לשרתים, NAS ותעבורה גבוהה במיוחד.
    40G Ethernet 40,000 \~38,000–39,000 מיועד לחוות שרתים ותשתיות ארגוניות.
    100G Ethernet 100,000 \~95,000–98,000 נמצא בשימוש אצל ספקי אינטרנט ו־Data Centers.
