    📘 מדריך שדרוג Nakivo Backup & Replication (Director ו־Transporters)

    מדריך זה מפרט את תהליך השדרוג של גרסת Nakivo Backup & Replication, כולל רכיב ה־Director וכל ה־Transporters, תוך שימוש ב־GUI או CLI.

    🔹 שלב 1: גיבוי ובדיקת הסביבה

    ✅ לפני כל שדרוג, בצע את הפעולות הבאות:

    • גבה את שרת ה־Director (באמצעות Snapshot ל־VM או Export אם מדובר במכונה פיזית)
    • ודא שיש גיבוי למערכת (Export Configuration + Master Password אם מוגדר)
    • אם מותקנים Transporters נוספים – ודא שהם מגובים גם כן
    • ודא שאין משימות גיבוי פעילות או מתוזמנות בזמן השדרוג

    🔹 שלב 2: בדיקת הגרסה הנוכחית

    • התחבר ל־Web UI של Nakivo
    • עבור אל Settings → About ובדוק את הגרסה (לדוגמה: 10.11.2.83985)
    • אתר את הגרסה העדכנית בדף ההורדות: https://www.nakivo.com/resources/download/

    🔹 שלב 3: שדרוג ה־Director

    🖥️ GUI

    1. התחבר ל־Web UI
    2. עבור אל Settings → Software Update
    3. לחץ על Check for Updates
    4. אם קיימת גרסה חדשה – תופיע אפשרות Update
    5. אשר את השדרוג והמתן לסיום (הממשק יתבצע אתחול)

    🖥️ CLI (Linux)

    # עצירת השירות
sudo systemctl stop nakivo-director

# הורדת קובץ ההתקנה (התאם את הגרסה)
wget https://download.nakivo.com/res/product/10.11.2.83985/NBR_Installer.sh

# הפיכת הקובץ להרצה
chmod +x NBR_Installer.sh

# הרצת ההתקנה
sudo ./NBR_Installer.sh

    🔹 שלב 4: שדרוג Transporters

    🛑 חשוב: גרסת ה־Transporter חייבת להיות תואמת לגרסת ה־Director. אין לשדרג Transporter לפני Director!

    🖥️ GUI

    1. התחבר ל־Web UI
    2. עבור אל Settings → Nodes
    3. אתר את ה־Transporter שלא מעודכן
    4. לחץ על סמל שלוש הנקודות ⋮ ובחר Upgrade

    🖥️ CLI (Linux)

    # עצירת השירות
sudo systemctl stop nakivo-transporter

# הורדת גרסת ההתקנה המתאימה
wget https://download.nakivo.com/res/product/10.11.2.83985/Nakivo_Transporter-Linux-x86_64.sh

# הפיכת הקובץ להרצה
chmod +x Nakivo_Transporter-Linux-x86_64.sh

# הרצת ההתקנה
sudo ./Nakivo_Transporter-Linux-x86_64.sh

    📦 QNAP NAS

    1. הורד את קובץ ה־.qpkg המתאים מהאתר: https://www.nakivo.com/resources/download/
    2. התחבר ל־QNAP > App Center
    3. בחר Install Manually והעלה את הקובץ
    4. המתן לסיום ההתקנה

    🔹 שלב 5: אימות לאחר שדרוג

    • ודא שכל ה־Transporters מופיעים תחת Settings → Nodes
    • בדוק שכולם עם גרסה זהה לזו של ה־Director
    • הפעל גיבוי בדיקה (Test Job) לוודא תקינות מלאה

    🧪 אימות ב־CLI ש־Transporter פעיל (QNAP / Linux)

    בדיקת תהליך רץ:

    ps | grep transporter

    פלט תקין לדוגמה:

    6276 admin  9644 S   /share/CACHEDEV1_DATA/.qpkg/NBR/transporter/bhsvc -r 9446

    בדיקת פורט פתוח (9446):

    netstat -tunlp | grep 9446
# או:
ss -tunlp | grep 9446

    בדיקת קישוריות מה־Director ל־Transporter:

    curl -vk https://<transporter-ip>:9446

    אם הכל תקין, תראה תקשורת TLS מתחילה (או שגיאת אישור – זה בסדר אם אין תעודת SSL).

    🔹 ווידוא דרך GUI

    • התחבר ל־Web UI של Nakivo (לדוגמה: https://192.0.2.10:4443)
    • עבור ל־Settings → Nodes
    • ודא שה־QNAP מופיע כ־Transporter עם כתובת ה־IP הנכונה (למשל 192.0.2.50)

    🔹 שינוי שם Transporter ב־GUI

    1. עבור ל־Settings → Nodes
    2. לחץ על שלוש הנקודות ליד שם ה־Transporter
    3. בחר Edit
    4. שנה את שם ה־Node (למשל "Backup NAS")
    5. לא פעיל? ניתן לבחור Remove ולמחוק את ה־Node

    📝 הערות חשובות

    • אין לשדרג Transporter לפני שדרוג Director
    • ודא שאתה מוריד את הגרסאות התואמות לארכיטקטורה (x86_64 / ARM)
    • ניתן לבצע Export Configuration לפני השדרוג (כולל הגדרות, משימות, פרטי גישה)
    • ניתן להשתמש ב־Onboard Transporter של ה־Director אם אין צורך ב־Transporter חיצוני
