    מדריך מלא: שליחת מיילים דרך Exchange Online (Microsoft 365) – כל האפשרויות והגדרות מעשיות

    מדריך זה מחולק לשני חלקים: 1. הסברים תיאורטיים על אפשרויות השליחה 2. 📘 הגדרות מעשיות בכל אחד מהשירותים בתוך Microsoft 365

    🛠️ חלק ב: מדריך מעשי – הגדרה ב־Microsoft 365 לכל שיטה

    🔹 1. SMTP Client Submission (smtp.office365.com)

    📌 תרחיש נפוץ:

    שליחה מאתר אינטרנט, אפליקציה, סקריפט, או שירות בענן באמצעות תיבת מייל קיימת ב־365

    🔧 שלבים לביצוע:

    1. בדוק שה־SMTP AUTH פעיל למשתמש: 

      Set-CASMailbox user@domain.com -SmtpClientAuthenticationDisabled $false

    2. אם מופעל MFA למשתמש:

    3. היכנס ל־https://mysignins.microsoft.com → צור App Password

    4. הגדרת שליחה במערכת חיצונית:

    5. SMTP Server: smtp.office365.com
    6. Port: 587
    7. TLS: Yes (STARTTLS)
    8. Username: user@domain.com
    9. Password: סיסמה רגילה או App Password

    🧠 הערה:

    • ניתן להשתמש בחשבון מערכת ייעודי (כגון noreply@)
    • רצוי לא להעניק הרשאות מלאות – רק הרשאות שליחה (אם צריך לשלוח בשם מישהו אחר)

    🔹 2. Direct Send (שליחה ישירה דרך MX של ה־365)

    📌 תרחיש נפוץ:

    שליחה ממדפסת, מערכת ERP או מצלמה רק למשתמשים פנימיים בארגון (אותו דומיין)

    🔧 שלבים ב־Microsoft 365:

    אין צורך בהגדרה בצד ה־365.

    הגדרה במכשיר:

    • SMTP Server: tenantname-com.mail.protection.outlook.com
    • Port: 25
    • TLS: אופציונלי (רצוי)
    • ללא אימות (no username/password)

    📌 דרישות:

    • תיבת הדוא"ל המקבלת חייבת להיות בתוך Microsoft 365 תחת הדומיין שלך
    • אין שליחה לחוץ לארגון

    🧠 טיפ:

    • כתובת MX המדויקת מתקבלת דרך Admin Center → Domains → domain.com → DNS Records

    🔹 3. SMTP Relay דרך Exchange Online עם Connector

    📌 תרחיש נפוץ:

    שרת ERP פנימי שולח מיילים החוצה דרך Microsoft 365 בלי צורך באימות יוזר

    🔧 הגדרת Connector:

    1. כניסה ל־Exchange Admin Center → Mail Flow → Connectors

    2. לחץ על ➕ New Connector

    3. הגדר:

    4. From: Your organization’s mail server
    5. To: Office 365
    6. Name: ERP SMTP Relay
    7. Authentication: Based on IP address
    8. הוסף את כתובת ה־IP הציבורית של השרת השולח

    9. אפשר TLS

    10. שמור וסיים את ההגדרה.

    📌 הגדרה במערכת (ERP, מדפסת וכו):

    • SMTP Server: tenantname-com.mail.protection.outlook.com
    • Port: 25
    • אין אימות (אין שם משתמש וסיסמה)
    • TLS: כן

    📌 דרישות נוספות:

    • ודא שרשומת SPF כוללת את ה־IP השולח או רק את include:spf.protection.outlook.com

    📘 מדריך הפעלת SPF, DKIM, DMARC בתוך Microsoft 365

    ✅ SPF

    1. כניסה לפאנל ניהול DNS של הדומיין
    2. הוסף רשומת TXT: 
      v=spf1 include:spf.protection.outlook.com -all
      או הוסף IP ישיר אם נדרש: 
      v=spf1 ip4:203.0.113.10 include:spf.protection.outlook.com -all

    ✅ DKIM

    1. כניסה ל־Microsoft 365 Defender → Email & Collaboration → Policies & Rules → DKIM
    2. הפעל DKIM לדומיין
    3. הוסף 2 רשומות CNAME בפאנל DNS כפי שמוצג לך
    4. לחץ שוב "Enable" לאחר שה־DNS מתעדכן

    ✅ DMARC

    1. פתח את ניהול ה־DNS
    2. הוסף רשומת TXT: 
      Name: _dmarc
Value: v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com

    🔍 בדיקת SPF, DKIM, DMARC – כלים ופקודות

    ✅ בדיקת SPF

    🔹 בדיקת רשומת SPF בדומין:

    nslookup -type=txt yourdomain.com

    חפש שורה שמתחילה ב׳ v=spf1 — לדוגמה:

    "v=spf1 ip4:203.0.113.15 include:spf.protection.outlook.com -all"

    🔹 בדיקת SPF בפועל של שליחה:

    • השתמש בכלי כגו: https://dmarcian.com/spf-survey/

    ✅ בדיקת DKIM

    🔹 בדיקת רשומת DKIM:

    nslookup -type=txt selector1._domainkey.yourdomain.com

    החלף selector1 בסלקטור שנמצא במיקרוסופט 365 (ברירת מחדל: selector1 או selector2)

    🔹 בדיקה מקוונת:

    • https://dmarcian.com/dkim-inspector/

    ✅ בדיקת DMARC

    🔹 בדיקת רשומת DMARC:

    nslookup -type=txt _dmarc.yourdomain.com

    🔹 דוגמה לפלט תקין:

    "v=DMARC1; p=reject; rua=mailto:dmarc-reports@yourdomain.com"

    ✅ בדיקת חתימות בפועל (SPF/DKIM/DMARC)

    • שלח מייל לבדיקה אל:
    • check-auth@verifier.port25.com
    • או https://www.mail-tester.com

    תקבל מייל חוזר עם תוצאה מלאה: SPF pass/fail, DKIM pass/fail, DMARC pass/fail

    🛠️ כלים מומלצים

    • https://mxtoolbox.com — בדיקות MX, SPF, DKIM, רשימות שחורות
    • https://dmarcian.com — ניתוח DMARC ו֤ SPF
    • https://mail-tester.com — בדיקת שליחה מלאה

    📥 הרשאות שליחה בשם משתמש מסוים (Send As או Send on Behalf)

    הגדרה ב־PowerShell:

    📌 שליחה בשם (Send on Behalf):

    Set-Mailbox user2@domain.com -GrantSendOnBehalfTo user1@domain.com

    📌 שליחה כ־(Send As):

    Add-RecipientPermission -Identity user2@domain.com -Trustee user1@domain.com -AccessRights SendAs

    שימושי כש:

    • מערכת צד שלישי שולחת בשם משתמש 365 קיים
    • מוניטין הדוא"ל חשוב (חתימה תקינה)
