    מדריך: התקנת LinOTP על שרת Ubuntu עם חיבור ל־Active Directory

    🎯 מטרת המדריך

    להתקין ולחבר את LinOTP על שרת Ubuntu כך שיבצע אימות מול Active Directory, כולל ניהול קודי OTP (כגון Google Authenticator), לצורך שימוש מול pfSense או כל פתרון RADIUS אחר.

    דרישות מקדימות

    • שרת Ubuntu 20.04 או 22.04 (מומלץ VM נפרדת)
    • גישת root
    • שרת Active Directory זמין (לדוגמה: dc.example.local)
    • חיבור לאינטרנט

    שלב 1: עדכון מערכת והתקנת תלותיות

    sudo apt update && sudo apt upgrade -y
sudo apt install -y apache2 libapache2-mod-wsgi-py3 python3-pip python3-venv postgresql

    שלב 2: התקנת LinOTP דרך Python virtualenv (שיטה עצמאית)

    1. צור סביבת עבודה: 

      mkdir /opt/linotp && cd /opt/linotp
python3 -m venv venv
source venv/bin/activate

    2. התקן את LinOTP: 

      pip install linotp

    3. הפעל את בסיס הנתונים: 

      sudo -u postgres createuser linotp
sudo -u postgres createdb -O linotp linotp

    שלב 3: הגדרת קובץ תצורה בסיסי

    1. צור את הקובץ /etc/linotp/linotp.ini: 

      [DEFAULT]
debug = true

[LinOTP]
sql.uri = postgresql://linotp:סיסמה@localhost/linotp

    2. צור קובץ Apache בשם /etc/apache2/sites-available/linotp.conf: 

      <VirtualHost *:80>
    WSGIDaemonProcess linotp user=www-data group=www-data threads=5 python-home=/opt/linotp/venv
    WSGIScriptAlias / /opt/linotp/venv/lib/python3.8/site-packages/linotp/wsgi.py

    <Directory /opt/linotp/venv/lib/python3.8/site-packages/linotp>
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/linotp_error.log
    CustomLog ${APACHE_LOG_DIR}/linotp_access.log combined
</VirtualHost>

    3. הפעל את השירות: 

      sudo a2ensite linotp
sudo systemctl reload apache2

    בדוק: http://<ip-server> — תופיע ממשק ניהול של LinOTP.

    שלב 4: הגדרת ממשק ניהול ובסיס נתונים

    1. פתח את LinOTP ב־דפדפן
    2. צור סיסמת admin ראשונית (אם לא נדרש - תמשיך)
    3. עבור ללשונית UserIdResolvers והוסף resolver מסוג LDAP:
    4. Name: AD
    5. URI: ldap://dc.example.local
    6. BindDN: משתמש בעל הרשאות קריאה ב־AD
    7. BaseDN: לדוגמה DC=example,DC=local
    8. Scope: Subtree

    שלב 5: יצירת מדיניות ושיוך טוקנים

    1. עבור ללשונית Policies > Add
    2. Scope: login
    3. Action: authenticate
    4. Realm: AD

    5. Users: *

    6. עבור ל־Tokens > Enroll Token

    7. בחר את המשתמש
    8. בחר סוג: TOTP
    9. סרוק את קוד ה־QR עם אפליקציית Google Authenticator

    שלב 6: התקנת רכיב RADIUS וחיבור ל־pfSense

    1. התקן את רכיב LinOTP RADIUS: 

      pip install linotp-radius

    2. הגדר את radius.ini: 

      [LINOTP]
LinOTPConfig = /etc/linotp/linotp.ini
SSLCheck = false
Realm = AD

    3. הפעל את RADIUS: 

      sudo systemctl restart freeradius

    4. ב־pfSense:

    5. עבור ל־System > User Manager > Authentication Servers
    6. הוסף שרת מסוג RADIUS
    7. כתובת: כתובת ה־LinOTP
    8. פורט: 1812
    9. פרוטוקול: PAP
    10. Shared Secret: תואם להגדרה ב־clients.conf של RADIUS

    סיום

    שרת LinOTP מותקן ומוכן לאימות מול Active Directory, עם ניהול קודי TOTP לכל משתמש, ומוגדר כשרת RADIUS תואם pfSense. מכאן תוכל להרחיב עם פורטל משתמשים, שכבות הרשאה לפי קבוצות ועוד.
